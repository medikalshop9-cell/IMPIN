## Three-Layer Architecture (always keep in mind)

**Layer 1 — Live Index:**
- Source: `data/raw/scraped_combined.csv` (1,586 products, 5 sources, May 2026)
- Script: `pipeline/build_index.py`
- Output: IMPIN snapshot — weighted food price index, base Jan-2026=100
- CPI basket weights: Food & Beverages 42.5%, General 40.5%, Household 9.8%, Clothing/Personal Care 7.2%

**Layer 2 — Nowcast:**
- Source: `data/processed/historical_panel.parquet` (48 months: wfp_food_index, ghsusd, brent)
- Best model: ARIMAX(1,1,0) — directional accuracy 83.3%
- Purpose: Estimate current month food CPI before GSS releases it
- Validation: compare model estimate for May 2026 vs IMPIN live snapshot

**Layer 3 — Forecast (1–3 months ahead):**
- Same historical panel + models
- Compare: ARIMAX vs XGBoost vs Random Forest vs Prophet
- Report: directional accuracy + RMSE on held-out test (2022-07 → 2023-07)

## Coding Standards

### Scrapers
- Every scraper must subclass `BaseScraper` (`scrapers/base.py`)
- Randomised delay 2–6 s between requests — never remove
- Handle HTTP 403/429 with exponential back-off (max 3 retries)
- Output columns: `source, cpi_category, product_name, price_ghc, unit, currency, url, scraped_at`
- Never hardcode credentials

### Models
- Save model artefacts to `models/results/` with `joblib`
- Always report RMSE, MAE, and directional accuracy on test set
- Train/test split: train up to 2022-06, test 2022-07 → 2023-07 (same as ARIMAX)
- Feature set for ML: lag features (t-1, t-2, t-3), log_ghsusd, brent_lag6, month dummies

### Statistical Analysis
- Run ADF + KPSS before any time-series modelling (already done — all I(1))
- First-difference before VAR/ML: Δ(wfp), log-return(ghsusd), Δ(brent)
- Granger at lags 1–8 with Bonferroni correction

### Dashboard
- Entry point: `app/dashboard.py`
- Use `st.session_state` for all interactive state
- 3 tabs: Live Index | Forecast | Anomalies
- Forecast chart must include confidence interval shading
